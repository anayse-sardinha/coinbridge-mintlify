To ensure a successful integration of the CoinBridge SDK on iOS, your application must adhere to the following security, technical, and design standards:


Before integration, contact the CoinBridge team to verify your account. 
Additionally, ensure the following parameters are provided:

* **Application Identity:** App Name, ADAM ID / Apple ID / App ID.
* **Development Details:** Team ID, Bundle ID, and AppGroups.
* **Security/APNs:** APNs Key ID and APNs Key File (.p8).

**User Side:** For the SDK to function securely within Apple Wallet, users must have **Face ID, Touch ID, or a passcode** enabled on their device.

## Security & Compliance Policies
Your application must implement these specific security measures to comply with financial regulations:

* **Multi-Factor Authentication (MFA):** Requires both a complex password and an OTP (SMS or Email).
* **Account Lockout:** Accounts must automatically lock after **5 consecutive failed login attempts**.
* **Feature Management:** * **Feature Flags:** A server-side toggle to enable/disable "Add to Apple Wallet" remotely.
    * **Force Update:** A mandatory mechanism to push users to the latest app version for security patches.

* [Add to Apple Wallet Button Guidelines](https://developer.apple.com/wallet/add-to-apple-wallet-guidelines/)

### Digital Card Art Specifications
Digital card images provided to CoinBridge must follow these exact layout rules:

| Attribute | Requirement |
| :--- | :--- |
| **Format** | Vector PDF (recommended) or raster PNG |
| **Resolution** | 1536 x 969 px |
| **Orientation** | Landscape with squared corners |
| **App Icon** | 100 x 100 px PNG (no text allowed) |

<Info> **Note on Layout:** Do not include physical card elements like card numbers, holograms, or chips. Apple will automatically overlay the last 4 digits (FPAN suffix) in the bottom-left corner of the image. </Info>
