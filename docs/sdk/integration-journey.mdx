Integrating CoinBridge has two sides: your **mobile app** (handled by your mobile developer) and your **server** (handled by your backend developer). Both sides need to be ready before you can go live.

## Milestones

<Steps>
  <Step title="Setup">
    *Mobile + Backend*

    Before writing any code, both tracks need their environments ready.

    The **mobile side** installs the SDK and configures the app with the right permissions (NFC, internet, biometrics).

    The **backend side** sets up a server that can receive and respond to CoinBridge requests quickly (under 400ms).

    Both teams can start this at the same time.
  </Step>

  <Step title="User Login + App Launch">
    *Mobile*

    The SDK needs to know who the user is every time the app opens. The mobile side connects your existing login system to the SDK so the user is recognised automatically.
  </Step>

  <Step title="Card Creation">
    *Mobile + QA*

    This step creates a digital card for the user and adds it to their Apple Wallet or Google Pay. Once this is done, the user can tap and pay.

    Your QA team tests this flow end-to-end in a test environment before it reaches real users.
  </Step>

  <Step title="Payment Authorization">
    *Backend*

    When a user taps to pay, CoinBridge calls your server to check if the transaction should go through. The backend side builds the endpoints that handle these checks.

    These calls happen in real time, so your server must respond in under 400ms.
  </Step>

  <Step title="Transaction Reporting">
    *Backend*

    After each payment, CoinBridge sends your server a report confirming what happened: approved, declined, or refunded. The backend side builds the endpoints that receive these reports.
  </Step>

  <Step title="Testing">
    *QA Engineer (+ Developer support)*

    Before going live, your QA team runs the full flow in a test environment: login, card setup, payments, refunds, and error cases.
  </Step>

  <Step title="CoinBridge Review">
    *Project Manager*

    <Warning>
      You cannot go live without CoinBridge approval. Submit your test results and wait for confirmation before proceeding.
    </Warning>

    You submit the test results to CoinBridge. They review and issue production credentials when everything looks good.
  </Step>

  <Step title="Go Live">
    *Mobile + Backend + Project Lead*

    Swap test credentials for live ones, run a final check, and turn the feature on for your users.
  </Step>
</Steps>
