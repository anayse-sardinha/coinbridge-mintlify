## System Architecture

The diagram below shows how each component interacts: from card setup through to a completed payment.

```mermaid
sequenceDiagram
    participant App as Mobile App
    participant SDK as CoinBridge SDK
    participant CB as CoinBridge Backend
    participant Cust as Your Backend
    participant Wallet as Device Wallet
    participant Net as Visa / Mastercard

    rect rgba(255, 253, 231, 0.8)
        Note over App,Net: Card Setup — One-Time Onboarding
        App->>SDK: Authenticate (JWT)
        SDK->>CB: Verify User
        opt Eligibility Check
            CB->>Cust: Get User Eligibility
            Cust-->>CB: Valid / Invalid
        end
        CB-->>SDK: Auth Confirmed
        App->>SDK: Issue Card
        SDK->>CB: Issue Card Request
        CB-->>SDK: Card Credential
        SDK->>Wallet: Provision Token
        Wallet-->>SDK: Token Active
    end

    rect rgba(255, 236, 179, 0.8)
        Note over App,Net: Payment — Per Transaction
        Wallet->>Net: NFC Tap
        Net->>CB: Authorization Request
        CB->>Cust: Check Balance
        Cust-->>CB: Available Balance
        opt Policy Check
            CB->>Cust: Verify Policies
            Cust-->>CB: Policy Decision
        end
        CB-->>Net: Approve / Decline
        CB-->>App: Push Notification
    end

    link SDK: Authentication @ /docs/sdk/installation/authentication
    link SDK: Issue Card @ /docs/sdk/installation/issue-card
    link SDK: Add to Wallet @ /docs/sdk/installation/add-to-wallet
    link Cust: User Eligibility @ /docs/webhooks/endpoints/get-user-details
    link CB: Payment Lifecycle @ /docs/payments/payments
```

| Symbol | Meaning |
|:---|:---|
| `——→` | Initiates a request or action |
| `╌╌→` | Response or async notification |
| `opt` | Optional step — may not apply to all integrations |
