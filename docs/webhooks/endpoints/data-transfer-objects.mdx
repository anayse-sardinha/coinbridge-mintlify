This page documents the shared request and response schemas used across the Backend Connector endpoints. All monetary amounts use floating-point numbers; all currency codes follow ISO 4217.

---

## EndUserDetailsDTO

Returned by your backend in response to the [Get User Eligibility](/docs/webhooks/endpoints/get-user-details) endpoint.

<ResponseField name="valid" type="boolean" required>
  Whether the user is permitted to onboard or use CoinBridge services. If `false`, the authentication process fails and the user is barred from proceeding.
</ResponseField>

```json Example
{
  "valid": true
}
```

---

## EndUserBalanceDTO

Returned by your backend in response to the [Get User Balance](/docs/webhooks/endpoints/get-user-balance) endpoint.

<ResponseField name="amount" type="float" required>
  The current net available funds for the user.
</ResponseField>

<ResponseField name="currencyCode" type="string" required>
  3-letter ISO 4217 currency code (e.g., `USD`, `EUR`, `ILS`).
</ResponseField>

<ResponseField name="validUntil" type="string (ISO 8601)">
  Optional expiration timestamp for this balance snapshot. Transactions received after this time will be rejected without a new balance fetch.
</ResponseField>

```json Example
{
  "amount": 115.75,
  "currencyCode": "EUR",
  "validUntil": "2025-12-22T14:02:28.137Z"
}
```

## TransactionAuthorizationDTO

Sent by CoinBridge to your backend via the [Authorize Transaction](/docs/webhooks/endpoints/authorize-transaction) endpoint.

<ResponseField name="reportId" type="string" required>
  Unique request identifier. Use for idempotency.
</ResponseField>

<ResponseField name="userId" type="string" required>
  Identifier for the user attempting the transaction.
</ResponseField>

<ResponseField name="details" type="object" required>
  Detailed transaction metadata including merchant ID, MCC, country code, and amount.
</ResponseField>

---

## TransactionAuthorizationResDTO

Returned by your backend in response to the [Authorize Transaction](/docs/webhooks/endpoints/authorize-transaction) endpoint.

<ResponseField name="authId" type="string" required>
  Authorization reference ID generated by your system. Returned to CoinBridge for audit tracking.
</ResponseField>

<ResponseField name="decision.approved" type="boolean" required>
  Whether the transaction is permitted based on user balance and spending rules.
</ResponseField>

<ResponseField name="decision.declineReason" type="string">
  Required if `approved` is `false`. A human-readable explanation shown to the user (e.g., `"Insufficient funds"`, `"Merchant not allowed"`).
</ResponseField>

```json Example: Approved
{
  "authId": "auth_abc123",
  "decision": {
    "approved": true
  }
}
```

```json Example: Declined
{
  "authId": "auth_abc124",
  "decision": {
    "approved": false,
    "declineReason": "Insufficient funds"
  }
}
```

## RefundReportDTO

Sent by CoinBridge to your backend via the [Report Refund](/docs/webhooks/endpoints/report-refund) endpoint.

<ResponseField name="reportId" type="string" required>
  Unique identifier for this refund event. Use for idempotency.
</ResponseField>

<ResponseField name="userId" type="string" required>
  The user whose account must be credited.
</ResponseField>

<ResponseField name="originalTransactionId" type="string">
  The ID of the original purchase transaction this refund is reversing.
</ResponseField>

<ResponseField name="refundDetails" type="object" required>
  Financial and merchant details for the refund, including amount and currency.
</ResponseField>

---

## CorrectionReportDTO

Sent by CoinBridge to your backend via the [Report Correction](/docs/webhooks/endpoints/report-correction) endpoint.

<ResponseField name="correctionReason" type="string" required>
  The reason for the adjustment. One of: `TRANSACTION_CANCELLED`, `SETTLEMENT_DIFF`, `TRANSACTION_EXPIRED`, `SETTLEMENT_POST_TRANSACTION_EXPIRATION`.
</ResponseField>

<ResponseField name="correctionAmount" type="float" required>
  Adjustment value. Positive credits the user; negative debits the user.
</ResponseField>

<ResponseField name="correctionCurrency" type="string">
  ISO 4217 currency code for the correction amount.
</ResponseField>

```json Example
{
  "correctionReason": "SETTLEMENT_DIFF",
  "correctionAmount": -3.50,
  "correctionCurrency": "USD"
}
```

## ReportResDTO

Returned by your backend to acknowledge any reporting endpoint (`/transaction`, `/refund`, `/correction`, `/user-event`).

<ResponseField name="eventId" type="string">
  Your internal tracking ID for the acknowledged event. Returned to CoinBridge for audit correlation.
</ResponseField>

```json Example
{
  "eventId": "evt_internal_789"
}
```
