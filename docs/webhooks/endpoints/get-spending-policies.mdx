---
title: "Get Spending Policies"
description: "Retrieve real-time spending rules and restrictions for a specific user."
api: "GET /coinbridge/api/v1/user/{userId}/rules"
---

The **Get Spending Policies** endpoint is an optional data-fetching service used to retrieve spending rulesets managed via your platform. This endpoint allows for real-time policy enforcement that can vary per user.

This call is triggered by the CoinBridge Backend during the payment authorization process. Upon successful retrieval of these rules (along with balance information), the backend proceeds with transaction authorization.

### Performance Requirements
As a time-critical data-fetching endpoint, this must meet strict performance standards to prevent service failures:
* **Response Time**: Up to 400ms.
* **SLA**: 99.96% monthly uptime.

## Path Parameters

<ParamField path="userId" type="string" required>
  The unique identifier of the logged-in user in your platform. This value must be recognized across all your application systems.
</ParamField>

## Response Body

<ResponseField name="rules" type="array" required>
  An array of spending rule objects to be applied to the user's current transaction attempt.
  
  <Expandable title="Supported Rule Objects">
    * **allowOnlyMccCodes**: (string[]) Only transactions with these Merchant Category Codes will be approved.
    * **restrictMccCodes**: (string[]) Transactions with these MCCs will be declined.
    * **allowOnlyMerchantIds**: (string[]) Only transactions with these specific MIDs will be approved.
    * **restrictMerchantIds**: (string[]) Transactions with these MIDs will be declined.
    * **allowOnlyCountryCodes**: (string[]) Only transactions from these 3-letter ISO-3166 countries will be approved.
    * **restrictCountryCodes**: (string[]) Transactions from these countries will be declined.
    * **maxTransactionCardAmount**: (float) Transactions above this amount (inclusive) will be declined.
    * **minTransactionCardAmount**: (float) Transactions below this amount (inclusive) will be declined.
  </Expandable>
</ResponseField>

<ResponseExample>
```json
{
  "rules": [
    {
      "restrictMccCodes": ["1731", "1520"]
    },
    {
      "maxTransactionCardAmount": 1000.00
    },
    {
      "restrictCountryCodes": ["AFG", "SYR"]
    }
  ]
}
```
</ResponseExample>

## Security
Authentication is based on a JSON Web Token (JWT) signed using the RS256 algorithm. CoinBridge provides an RSA public key to your backend developers to verify each request.
